1) GIT — COMPLETE WORKFLOW & COMMANDS
============================================================

------------------------------------------------------------
A) REPOSITORY INITIALIZATION & FIRST PUSH
------------------------------------------------------------
git init
git status
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin <repo-url>
git push -u origin main

------------------------------------------------------------
B) CLONE / FETCH / PULL / CHECKOUT
------------------------------------------------------------
git clone <repo-url>
git pull
git fetch origin
git fetch --all
git checkout <branch-name>
git checkout -b <new-branch>

------------------------------------------------------------
C) WORKING WITH BRANCHES
------------------------------------------------------------
git branch
git branch feature/<feature-name>
git checkout feature/<feature-name>
git merge feature/<feature-name>
git push origin feature/<feature-name>

------------------------------------------------------------
D) DISCARDING / RESETTING CHANGES
------------------------------------------------------------
git restore <file>
git restore .
git reset HEAD <file>          (unstage but keep changes)
git reset --hard               (discard everything)

------------------------------------------------------------
E) STASHING
------------------------------------------------------------
git stash
git stash apply
git stash pop
git stash list

------------------------------------------------------------
F) PUSHING AFTER NON-FAST-FORWARD ERROR
------------------------------------------------------------
git pull --rebase origin <branch-name>
# resolve conflicts if needed
git push

------------------------------------------------------------
G) DELETE LOCAL & REMOTE BRANCH
------------------------------------------------------------
git branch -d feature/<name>
git branch -D feature/<name>
git push origin --delete feature/<name>

------------------------------------------------------------
H) CHANGE REMOTE URL
------------------------------------------------------------
git remote set-url origin <new-url>

------------------------------------------------------------
I) REMOVE A FILE ACCIDENTALLY COMMITTED
------------------------------------------------------------
git rm --cached <file>
git commit -m "remove sensitive file"
git push

------------------------------------------------------------
J) LOG & HISTORY
------------------------------------------------------------
git log
git log --oneline --graph --decorate
git diff
git blame <file>

------------------------------------------------------------
K) PUSH TO DIFFERENT REMOTE
------------------------------------------------------------
git remote add neworigin <repo-url>
git push neworigin main

2) COLLABORATIVE CODING USING GIT (TEAM WORKFLOW)
============================================================

------------------------------------------------------------
A) STANDARD TEAM WORKFLOW
------------------------------------------------------------
# Developer 1 creates ORG → Creates repository
# Other developers added as collaborators
git clone <repo-url>
git checkout -b feature/<task>
git add .
git commit -m "<work>"
git push origin feature/<task>
# Open Pull Request
# Wait for review + merge to main

------------------------------------------------------------
B) RESOLVE MERGE CONFLICTS
------------------------------------------------------------
git pull origin main
# Open conflicting files
# Edit around <<<<<<< ======= >>>>>> markers
git add .
git commit
git push

------------------------------------------------------------
C) PATCH CREATION AND APPLYING
------------------------------------------------------------
# Create patch
git format-patch -1 <commit-hash>
# Apply patch
git apply <patch-file>

3) MAVEN — PROJECT BUILDING & PACKAGING
============================================================

------------------------------------------------------------
A) BASIC BUILD COMMANDS
------------------------------------------------------------
mvn clean
mvn compile
mvn test
mvn package
mvn install
mvn deploy

------------------------------------------------------------
B) STRUCTURE (IMPORTANT FOLDERS)
------------------------------------------------------------
src/main/java       → main Java code
src/test/java       → unit tests
src/main/resources  → configuration
target/             → build output (.jar/.war)

------------------------------------------------------------
C) ADDING DEPENDENCIES (POM.XML)
------------------------------------------------------------
<dependencies>
    <dependency>
        <groupId>group</groupId>
        <artifactId>artifact</artifactId>
        <version>version</version>
    </dependency>
</dependencies>

------------------------------------------------------------
D) PARENT + MULTI-MODULE MAVEN PROJECT
------------------------------------------------------------
Parent POM has:
<packaging>pom</packaging>
<modules>
    <module>module1</module>
    <module>module2</module>
</modules>

Build complete project:
mvn clean install

------------------------------------------------------------
E) GENERATING EXECUTABLE JAR
------------------------------------------------------------
mvn clean package
# Run JAR
java -jar target/<name>.jar

8) JENKINS — MAVEN WEB AUTOMATION (BUILD → TEST → DEPLOY)
============================================================

------------------------------------------------------------
A) JOB 1: MavenWeb_Build
------------------------------------------------------------
SCM: Git repo
Build Steps:
  mvn clean
  mvn install
Post-Build:
  Archive artifacts → **/*
  Trigger other projects → MavenWeb_Test

------------------------------------------------------------
B) JOB 2: MavenWeb_Test
------------------------------------------------------------
Env:
  Delete workspace before build
Build Steps:
  Copy artifacts from MavenWeb_Build
  mvn test
Post-Build:
  Archive artifacts → **/*
  Trigger → MavenWeb_Deploy (only if stable)

------------------------------------------------------------
C) JOB 3: MavenWeb_Deploy
------------------------------------------------------------
Build Step:
  Copy artifacts from MavenWeb_Test
Post-Build:
  Deploy WAR/EAR to container
  WAR = **/*.war
  Context path = webpath
  Provide Tomcat URL + Credentials

9) JENKINS — SCRIPTED PIPELINE
============================================================
pipeline {
    agent any
    stages {
        stage('Checkout') { steps { git '<repo-url>' } }
        stage('Build')    { steps { sh 'mvn clean install' } }
        stage('Test')     { steps { sh 'mvn test' } }
        stage('Archive')  { steps { archiveArtifacts artifacts: "**/*" } }
    }
}
